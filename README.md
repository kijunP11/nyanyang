프로젝트명: 냐냥 플랫폼 개발

개발자(수급인):박기준 의뢰인(발주처):김보현
제1조 (목적)
본 계약은 의뢰인이 요청한 “냐냥 플랫폼”의 설계·개발·납품 및 안정화 지원을 규정함을 목적으로 한다.
제2조 (작업 범위)

1. MVP (3개월)
   ● 회원/인증: 이메일/소셜(구글·애플), 본인(성인)인증, 추천인 코드
   ● 홈/탐색: 출석체크, 공지/이벤트 배너, 검색/정렬
   ● 채팅방: 모델 선택, 대화 저장, 커스터마이징/테마, 롤백·분기, 재생성, 요약 메모리(기본)
   ● 캐릭터 제작/관리: 프로필·설정·미디어 업로드·키워드북, 세이프티 필터
   ● 마이페이지: 내 정보, 포인트 관리
   ● 결제: Stripe(기본) 카드 결제 연동
   ● Admin 기본: 유저 관리, 캐릭터 승인/삭제, 결제 조회, 공지 등록, 신고 처리 2.확장 (추가 2~3개월)
   ● 추천인/이벤트 고도화, 대화 공유/갤러리
   ● 크리에이터 모드(제작자 인증·정산·활동 관리)
   ● 국내 PG(카카오/네이버/토스) 연동
   ● Admin 고도화(통계·모니터링·자동 리포트·권한 분리)
   ● 보안 강화(운영자 2FA, 내부 접근 제한)
   제3조 (기간 및 마일스톤)
   ● MVP 개발 기간: 계약 체결일로부터 약 3개월
   ● 확장 기능 개발 기간: MVP 완료 후 약 2~3개월
   제4조 (대금 및 지급 방법)
   총 개발비용: \7,000,000
1. MVP 단계 (총 \4,700,000)
   ● 착수금 30% (\1,410,000) → 계약 체결 직후
   ● 1차 마일스톤 15% (\705,000) → 2주차 완료 (기본 세팅·DB·이메일 로그인)
   ● 2차 마일스톤 15% (\705,000) → 4주차 완료 (소셜 로그인·본인인증·홈/탐색)
   ● 3차 마일스톤 15% (\705,000) → 8주차 완료 (채팅 기본·캐릭터 관리)
   ● 코드프리즈 15% (\705,000) → 11주차 완료 (롤백/분기, 요약 메모리, 마이페이지)
   ● 잔금 10% (\470,000) → 12주차 검수 완료 및 납품
1. 확장 단계 (총 \2,300,000)
   ● 확장 착수 30% (\690,000) → 확장 계약 시작
   ● 확장 1차 25% (\575,000) → 16주차 완료 (추천인/이벤트 고도화, 갤러리)
   ● 확장 2차 25% (\575,000) → 20주차 완료 (크리에이터 모드, 국내 PG 연동)
   ● 확장 잔금 20% (\460,000) → 24주차 최종 납품 및 QA
   ※ 모든 대금은 계좌이체로 진행하며, 청구서 발행일로부터 영업일 3일 이내 지급.
   제5조 (검수 및 인수)
   ● 각 마일스톤 결과물 제출 후 영업일 5일 내 피드백이 없으면 조건부 인수로 간주한다.
   ● 치명적 결함 발생 시, 개발자가 우선 수정 후 인수 확정으로 한다.
   제6조 (유지보수)
   ● 납품 후 무상 안정화 30일 제공(버그 수정 포함).
   ● 무상 유지보수는 기획서/명세서 대비 불일치로 정의된 버그 수정까지만 포함한다.
   ● 기능 추가 및 장기 유지보수는 별도 계약(시간 단가 또는 월 단위)으로 진행한다.
   제7조 (운영비·제3자 비용)
   ● 애플/구글 개발자 계정, PG 수수료, 인증/문자 서비스, 클라우드 사용료(Supabase, Cloudinary 등)는 의뢰인 부담. ● 계정은 의뢰인 명의로 개설·결제하며, 개발자는 제공받은 API Key로 연동한다.
   ● 개발자가 선결제할 경우 반드시 의뢰인의 사전 서면 동의를 거쳐야 하며, 영수증 제출 후 실비 + 관리 수수료(5%)로 정산한다.
   제8조 (지식재산권 및 산출물 소유권)
   ● 대금 완납 시 전체 소스코드 및 산출물 소유권은 의뢰인에게 이전된다.
   ● 계약이 중도 해지되는 경우, 이미 지급한 금액에 해당하는 작업물은 즉시 제공하며, 미지급 부분은 정산 완료 후 제공한다.
   ● 오픈소스 및 외부 서비스는 해당 라이선스를 따른다.
   제9조 (데이터 소유권 및 이관)
   ● 모든 데이터 소유권은 의뢰인에 귀속된다.
   ● Supabase 등 벤더는 호스팅 제공자일 뿐 소유권은 이전되지 않는다.
   ● 대금 완납 후 7영업일 이내에 DB dump, 환경설정, 관리자 권한 등을 의뢰인에게 이관하며, 이관 완료 후 개발자의 접근 권한은 모두 회수한다.
   제10조 (버그 수정 기준)
   ● 버그는 다음과 같이 분류·대응한다.
   ○ 치명적: 48~72시간 내 착수, 필요 시 임시 우회 제공
   ○ 주요: 7영업일 내
   ○ 경미: 14영업일 내
   ● 신규 기능 요청은 버그가 아닌 별도 개발로 처리한다.
   제11조 (비밀유지)
   개발 과정에서 알게 된 모든 정보는 제3자에게 공개하지 않는다. 제12조 (중도 해지)
1. 계약 당사자 일방은 사전 서면 통지(7일 전)를 통해 계약을 해지할 수 있다.
1. 해지 시, 개발자는 해지일까지 완료된 산출물과 소스를 의뢰인에게 전달한다.
1. 의뢰인은 해지일까지 완료된 작업에 해당하는 비용을 정산한다.
1. 이미 지급된 착수금은 반환하지 않는다.
   제13조 (지체상금)
1. 개발 일정이 지연되는 경우, 영업일 5일의 유예기간을 둔다.
1. 불가항력 사유(예: 애플/구글 심사 지연, PG사 심사 지연, 클라우드 장애 등)는 지연 산정에서 제외한다. 단, 개발자 내부 인력·리소스 문제는 제외한다.
1. 유예기간 초과 시, 미지급 마일스톤 금액의 0.1%/일을 지체상금으로 산정하되, 총액은 계약금의 5%를 상한으로 한다.
1. 지연이 지속될 경우, 의뢰인은 해당 마일스톤 대금 지급을 보류하고, 수정 완료 후 지급할 수 있다.
   제14조 (하도급 금지)
   개발자는 의뢰인의 사전 동의 없이 제3자에게 본 계약상의 업무를 재위탁하지 않는다. 제15조 (분쟁 해결) 본 계약에 관한 분쟁은 서울중앙지방법원을 전속 관할로 한다.
